   110→- `Color.withOpacity(...)` is used in 7 places (`update_banner.dart`, `virent_ui.dart` ×6). Deprecated since Flutter 3.27 in favour of `Color.withValues(alpha: ...)`. Not a compile error, just a warning.

   112→### Other Issues Found
   113→- **`api_exceptions.dart:40`** — `ApiException.fromResponse` calls `Uri.splitQueryString(response.body)` to parse the body, but API error responses are JSON. Query-string parsing won't extract `error` from a JSON body; the try/catch silently swallows the failure and falls back to the raw body string. Functionally defensive but logically wrong.
   114→- **`main_web.dart:16`** — `routerProvider` constructs `StorageService()` but never calls `init()`. Works (lazy init via `_instance()`) but inconsistent with `main.dart` which calls `await storage.init()`.
   115→- **`main_web.dart`** — Does not start the embedded server and does not wrap with `UpdateBannerWrapper` (unlike `main.dart`). If web is intended to be a client-only mode, this is fine; otherwise incomplete.
   116→- **`app_constants.dart:57` vs `storage_service.dart:157`** — `AppConstants.firstRunKey = 'first_run'` duplicates `StorageKeys.isFirstRun = 'first_run'`. Two sources of truth for the same key.
   117→- **`embedded_server.dart`** is 1628 lines — a single monolith with 12 route-registration methods. Maintainability concern.
